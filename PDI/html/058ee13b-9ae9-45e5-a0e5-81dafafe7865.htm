      Version 1.2.0.3

      Version 1.2.0.3 was released on June 25th, 2005.

      Changes in This Release

      • Fixed a bug in the recurrence engine that could cause it to miss instances in the weekly, hourly, minutely, and secondly patterns on occasion.

      • Adjusted the parser for vCalendar 1.0 alarm properties to take into account that not all parameters may be present.

      • Added a NextInstance method to the Recurrence class to allow retrieval of the next recurrence instance on or after a specified date.

      • Modified the calendar objects so that under vCalendar 1.0 the objects no longer write out a unique ID unless one has been created. This makes it conform to the specification which does not require a unique ID. iCalendar 2.0 requires a unique ID and one will always be generated if necessary for each calendar object.
