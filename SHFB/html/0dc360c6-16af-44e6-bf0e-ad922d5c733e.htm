      Presentation Styles

      This topic covers common questions and issues related to presentation styles.

      Can presentation styles be configured?

      Not in the way that build components and plug-ins can be configured. However, most presentation styles do expose transformation arguments that can be modified using the Transform Args project property category.

      Why can't I create a help project that contains a combination of Open XML/Markdown and another format?

      Open XML and Markdown presentation styles do not generate HTML output like the other help file formats do. As such, they cannot be combined in a single project. Create a separate project to generate Open XML or Markdown help output.

      I want to modify aspects of the presentation style such as the style sheets used or make changes to the scripts. How do I do that?

      For minor changes to content, use the stock content file replacement ability of the help file builder. For more significant changes or changes that require modifications to the transformations, you will need to create your own presentation style or clone one of the existing ones.

      How can I create my own presentation style or clone one of the existing ones?

      For information on creating your own presentation styles, see the Creating a Presentation Style Component topic.

      How do I add a syntax section for a new language that doesn't appear in the list of current syntax filter generators?

      For information on creating your own syntax filter generators, see the Creating a Syntax Filter Generator Component topic.
