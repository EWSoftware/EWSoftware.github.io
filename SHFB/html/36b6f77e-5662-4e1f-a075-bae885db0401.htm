XSL Transform Tool

This tool, XslTransform.exe, is used to transform an XML file using one or more specified XSL transformations. It is used at various points in the build process to add entries to the reflection information file using additional XSL transformations, transform the reflection information file to produce the topic manifest for BuildAssembler, etc.

Command Line Parameters

This tool can be executed from the command line. The following parameters are available.

/xsl:xsltPath1[,xsltPath2,...]

Required. This is used to specify one or more XSL transformation files as a comma-separated list. The transformations will be applied to the target XML file in the order specified.

/arg:name=value[,name=value,...]

Optional. This is used to specify one or more XSL transformation arguments as a comma-separated list. The arguments must be in the form "name=value". They will be passed in to each XSL transformation.

/out:[outputFile]

Optional. This is used to specify the name of the file to which the transformed XML is written. If not specified, the output is written to the console.

/w

Optional. This is used to specify that insignificant whitespace should not be ignored. If omitted, insignificant whitespace is ignored.

xmlFile

Required. Specify the XML file that will be transformed.

Example Usage
Copy
XslTransform.exe /xsl:Transform1.xslt,Transform2.xslt /arg:Param1=true,Param2=Test
  /out:reflection.transformed reflection.xml
MSBuild Task Properties

This tool can be executed as an MSBuild task. The following properties are available.

WorkingFolder

Optional. This property is used to set the working folder. If not set, the current directory is used as the working folder.

InputFile

Required. This property is used to specify the XML file to transform.

OutputFile

Required. This property is used to set the output filename. This file will contain the transformed content from the input file.

Transformations

Required. This property is used to pass in the list of transformations to run. Separate multiple transforms with a semi-colon. Paths starting with "~/" or "~\" are assumed to refer to a Sandcastle transformation file and will be fully qualified with the root Sandcastle tools folder (the folder of the executing assembly). Relative paths are assumed to be relative to the working folder. Absolute paths are not modified.

Arguments

Optional. This property is used to pass in any optional XSL transform arguments. The arguments must be in the form "name=value". Separate multiple arguments with a semi-colon. They will be passed in to each XSL transformation.

Example Usage
Copy
<UsingTask TaskName="Microsoft.Ddue.Tools.MSBuild.XslTransform"
  AssemblyFile="C:\HelpFileBuilderTools\XslTransform.exe" />

<PropertyGroup>
  <WorkingFolder></WorkingFolder>
</PropertyGroup>

<Microsoft.Ddue.Tools.MSBuild.XslTransform
  WorkingFolder="$(WorkingFolder)"
  Transformations="~\Transforms\MergeDuplicates.xsl;~\Transforms\AddXamlSyntax.xsl"
  InputFile="reflection.org"
  OutputFile="reflection.xml" />
