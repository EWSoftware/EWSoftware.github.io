    Additional Content Support

    Additional content support is similar in nature to the NDoc properties AdditionalContentResourceDirectory, FilesToInclude, RootPageFileName and RootPageTOCName. However, the help file builder's implementation provides additional benefits over NDoc:

    • Files are added to the project directly.

    • The folder structure of the files in the project becomes their layout in the compiled help file. Files at the root of the project are copied to the root of the compiled help file. Files in subfolders are copied recursively to like-named subfolders in the compiled help file.

    • By adding a file to the project with a name that matches a stock content file in a folder that matches the folder of the stock content file, you can replace that stock content file. For example, by adding a folder called styles\ to the project and placing a file in that folder called Presentation.css, you can replace the default style sheet used by the selected presentation style.

    This implementation provides a great deal of flexibility and finer control over the inclusion of additional content items.
